# MVP сервиса тестирования VPN и мессенджеров

Django-сервис для сбора результатов тестирования VPN-приложений и мессенджеров.

Реализовано:

- регистрация и авторизация пользователей;
- роли и подтверждение пользователей администратором;
- организации, города, операторы связи и справочники;
- справочник VPN-приложений с периодами;
- VPN-анкеты, первый и второй замер;
- автоматический перевод завершенных VPN-замеров в статус отправки;
- анкеты мессенджеров с одним суточным замером;
- контроль полноты ежедневных VPN-анкет и анкет мессенджеров;
- PostgreSQL в Docker.

Telegram-интеграция, импорт/экспорт Excel для анкет и аналитические дашборды пока не реализованы.

## Стек

- Python 3.12
- Django
- PostgreSQL 16
- Docker Compose
- Gunicorn
- Django Templates
- Bootstrap 5
- openpyxl

## Структура

```text
config/          настройки проекта и корневые URL
accounts/        пользователи, роли, регистрация, подтверждение
references/      организации, города, операторы связи, VPN, мессенджеры
questionnaires/  VPN-анкеты, анкеты мессенджеров, замеры и результаты проверки
control/         ежедневный план ожидаемых анкет и контроль полноты сдачи
core/            главная страница и личный кабинет
templates/       HTML-шаблоны Bootstrap
static/          статические файлы
```

## Переменные окружения

Создайте `.env` из примера:

```bash
cp .env.example .env
```

На Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

Минимально проверьте значения:

```env
SECRET_KEY=change-me
DEBUG=False
ALLOWED_HOSTS=127.0.0.1,localhost
FORCE_SCRIPT_NAME=
USE_X_FORWARDED_HOST=True

POSTGRES_DB=vpn_testing
POSTGRES_USER=vpn_testing
POSTGRES_PASSWORD=change-me
POSTGRES_HOST=db
POSTGRES_PORT=5432
```

Реальный `.env` не добавляйте в репозиторий.

## Запуск через Docker Compose

Собрать и запустить PostgreSQL и Django:

```bash
docker compose up -d --build
```

Сервис Django будет доступен только локально:

```text
http://127.0.0.1:8010/
```

Nginx-контейнер не используется. Если нужен внешний доступ, проксируйте хостовым Nginx на `127.0.0.1:8010`.

Если сервис должен открываться не из корня домена, а с префиксом, например `http://83.229.14.119/testvpn/`, добавьте в `.env`:

```env
FORCE_SCRIPT_NAME=/testvpn
```

Пример блока Nginx для такого варианта:

```nginx
location /testvpn/static/ {
    alias /opt/vpn-testing/staticfiles/;
}

location /testvpn/ {
    proxy_pass http://127.0.0.1:8010/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location / {
    return 302 /testvpn$request_uri;
}
```

После изменения `.env` перезапустите Django:

```bash
docker compose restart web
```

## Миграции и статика

При старте `web` автоматически выполняет:

```bash
python manage.py migrate
python manage.py collectstatic --noinput
gunicorn config.wsgi:application --bind 0.0.0.0:8000
```

Вручную применить миграции:

```bash
docker compose exec web python manage.py migrate
```

Заполнить базовые справочники:

```bash
docker compose exec web python manage.py seed_initial_data
```

Создать администратора без интерактивного ввода:

```bash
docker compose exec web python manage.py create_admin --username admin --password "change-me"
```

На Windows лучше не вводить кириллицу в интерактивной консоли Docker. Русское ФИО администратора можно поменять позже через Django Admin в браузере.

Если хотите использовать переменные из `.env`, заполните:

```env
DJANGO_SUPERUSER_USERNAME=admin
DJANGO_SUPERUSER_PASSWORD=change-me
DJANGO_SUPERUSER_FULL_NAME=admin
```

И выполните:

```bash
docker compose exec web python manage.py create_admin
```

## Остановка

Остановить контейнеры без удаления данных:

```bash
docker compose down
```

Данные PostgreSQL хранятся в Docker volume `postgres_data`.

## Backup PostgreSQL

Создать дамп:

```bash
docker compose exec db pg_dump -U "$POSTGRES_USER" -d "$POSTGRES_DB" > backup.sql
```

В PowerShell удобнее явно указать значения из `.env`:

```powershell
docker compose exec db pg_dump -U vpn_testing -d vpn_testing > backup.sql
```

## Restore PostgreSQL

Восстановить дамп в существующую базу:

```bash
docker compose exec -T db psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" < backup.sql
```

PowerShell-вариант:

```powershell
Get-Content backup.sql | docker compose exec -T db psql -U vpn_testing -d vpn_testing
```

## Основные URL

- `/` - главная страница;
- `/accounts/register/` - регистрация;
- `/accounts/login/` - вход;
- `/accounts/users/` - управление пользователями для администратора;
- `/references/` - справочники;
- `/vpn/forms/` - список VPN-анкет;
- `/vpn/forms/create/` - создание VPN-анкеты;
- `/vpn/review/` - VPN-анкеты на проверку, временно отключено в пользовательском сценарии;
- `/messengers/forms/` - список анкет мессенджеров;
- `/messengers/forms/create/` - создание анкеты мессенджеров;
- `/messengers/review/` - анкеты мессенджеров на проверку;
- `/control/` - контроль полноты ежедневных анкет;
- `/control/<YYYY-MM-DD>/` - детализация плана на дату;
- `/control/<YYYY-MM-DD>/generate/` - формирование плана на дату для администратора;
- `/control/<YYYY-MM-DD>/missing/` - отсутствующие анкеты;
- `/control/<YYYY-MM-DD>/drafts/` - анкеты в черновике;
- `/control/<YYYY-MM-DD>/submitted/` - отправленные анкеты;
- `/control/<YYYY-MM-DD>/confirmed/` - подтвержденные анкеты;
- `/control/<YYYY-MM-DD>/returned/` - возвращенные анкеты;
- `/control/<YYYY-MM-DD>/export/` - Excel-выгрузка данных за дату из контроля;
- `/exports/` - страница экспорта с фильтрами;
- `/exports/excel/?date=<YYYY-MM-DD>` - скачивание Excel-файла;
- `/admin/` - Django Admin.

## Роли

- `tester` - создает и заполняет свои VPN-анкеты.
- `coordinator` - видит анкеты своей организации. Согласование и возврат временно отключены.
- `reader` - просматривает доступные данные без редактирования.
- `admin` - управляет пользователями, справочниками и анкетами.

Суперпользователь Django автоматически получает роль администратора и статус подтверждения.

## Проверка после запуска

1. Запустите контейнеры: `docker compose up -d --build`.
2. Выполните `docker compose exec web python manage.py seed_initial_data`.
3. Создайте администратора через `create_admin`.
4. Зайдите в `/admin/` и проверьте справочники.
5. Зарегистрируйте пользователя через `/accounts/register/`.
6. Подтвердите пользователя администратором.
7. Под тестировщиком создайте VPN-анкету.
8. Пройдите первый замер.
9. После заполнения последнего VPN замер автоматически перейдет в статус `Отправлен`.
10. Если заполнен только первый замер, анкета станет `Частично отправлена`; после второго замера анкета станет `Отправлена`.

## Проверка анкет мессенджеров

1. В Django Admin или разделе справочников создайте активные мессенджеры, например Telegram и WhatsApp.
2. Под тестировщиком откройте `/messengers/forms/` и создайте анкету.
3. Нажмите `Начать тестирование`.
4. Заполните результаты по каждому активному мессенджеру:
   - доступность сервиса;
   - отправку сообщения;
   - время отправки, если сообщение отправлено;
   - объем и скорость файла, если файл отправлялся;
   - аудиозвонок;
   - комментарий, если выбран статус `Невозможно проверить`.
5. На странице завершения отправьте анкету на проверку.
6. Под координатором откройте `/messengers/review/`.
7. Откройте анкету и подтвердите ее или верните с обязательным комментарием.
8. Проверьте ограничения:
   - тестировщик не может редактировать отправленную анкету, пока ее не вернули;
   - читатель видит анкеты, но не видит кнопок редактирования, подтверждения и возврата;
   - координатор видит только анкеты своей организации.

## Контроль полноты ежедневных анкет

Раздел `/control/` показывает план ожидаемых анкет на дату и фактическое состояние сдачи.

В ежедневный план входят:

- 56 VPN-анкет;
- 56 анкет мессенджеров.

ГРЧЦ в план веб-анкет не включается: эти данные предполагается импортировать отдельно из Excel.

Для организаций-операторов связи укажите поле `Связанная сим-карта` в организации:

- организация `МТС` -> сим-карта `МТС`;
- организация `МегаФон` -> сим-карта `МегаФон`;
- организация `Билайн` -> сим-карта `Билайн`.

Также для мессенджеров у организации должен быть указан основной город. Для ВУЗов город используется как закрепленный город организации. Для VPN операторов связи проверка планируется по всем активным городам.

Проверка сценария:

1. Выполните миграции: `docker compose exec web python manage.py migrate`.
2. Обновите справочники: `docker compose exec web python manage.py seed_initial_data`.
3. Под администратором откройте `/control/`.
4. Нажмите `Сформировать план` на нужную дату.
5. Проверьте, что создано 112 плановых записей при 4 ВУЗах, 3 организациях-операторах, 4 сим-картах и 2 ОС.
6. Создайте несколько VPN-анкет и анкет мессенджеров на эту дату.
7. Вернитесь в `/control/<YYYY-MM-DD>/` и проверьте переход статусов из `Отсутствует` в `Черновик`.
8. Подтвердите часть анкет и проверьте счетчики `Подтверждена` и процент готовности.
9. Под координатором проверьте, что видны только записи своей организации.
10. Под читателем проверьте просмотр без кнопки формирования плана.
11. Под тестировщиком проверьте, что раздел контроля недоступен.

## Экспорт Excel

Раздел экспорта формирует один плоский `.xlsx` файл для дальнейшего анализа. Файл создается в памяти и не сохраняется на сервере.

Доступ:

- администратор экспортирует все данные;
- читатель экспортирует все данные;
- координатор экспортирует только данные своей организации;
- тестировщик не видит кнопку экспорта и не имеет доступа к общему экспорту.

Файл содержит листы:

- `VPN` - одна строка на один результат VPN в конкретном замере;
- `Мессенджеры` - одна строка на один результат проверки мессенджера;
- `Контроль` - сводка готовности за выбранную дату и сводка по организациям.

Основные URL:

- `/exports/` - форма экспорта;
- `/exports/excel/?date=2026-05-26` - скачать файл за дату;
- `/control/2026-05-26/export/` - скачать файл со страницы контроля.

На странице `/exports/` сначала применяются фильтры, затем можно выбрать конкретные анкеты чекбоксами. Кнопка `Выбрать всё` отмечает все анкеты из текущей отфильтрованной выдачи, `Сбросить всё` снимает отметки. Если не выбрать ни одной анкеты, Excel формируется по всем данным, которые подходят под фильтр.

В листе `VPN` организация выводится в колонке `Тестировщик`; это тестирующая организация, например МТУСИ, ПГУТИ, СибГУТИ, Билайн, МТС. Результаты проверок VPN выгружаются в кодах: `Работает` -> `1`, `Не работает` -> `0`, прочерк остается `-`.

Имя файла:

- `testing_export_YYYY-MM-DD.xlsx` для одной даты;
- `testing_export_YYYY-MM-DD_YYYY-MM-DD.xlsx` для периода.

Готовность к выгрузке считается по статусам `submitted` и `confirmed`. Статус `partially_submitted` не считается полной готовностью. Если день не готов, экспорт все равно доступен администратору и читателю, но файл будет содержать неполные данные.

Проверка сценария:

1. Выполните миграции: `docker compose exec web python manage.py migrate`.
2. Сформируйте план на дату через `/control/`.
3. Создайте несколько VPN-анкет и заполните первый замер.
4. Проверьте, что VPN-анкета стала `Частично отправлена` или `Отправлена`.
5. Создайте и заполните несколько анкет мессенджеров.
6. Откройте `/control/<YYYY-MM-DD>/`.
7. Нажмите `Сформировать Excel`.
8. Проверьте, что файл содержит листы `VPN`, `Мессенджеры`, `Контроль`.
9. Под координатором проверьте, что в файле только его организация.
10. Под тестировщиком проверьте, что кнопка экспорта не отображается.
11. В Django Admin проверьте запись `ExportLog`.
